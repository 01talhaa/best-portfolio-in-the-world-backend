Endpoint: PUT /api/v1/clients/:id
Headers: 
  Content-Type: application/json
  Authorization: Bearer {access_token}

Request Body: (include only fields you want to update)
{
  "status": "active",

    "title": "Senior CTO",
    "email": "john.smith.jr@techcorp.com",
    "phone": "+1-555-0125"
  },
  "description": "Updated company description with recent achievements..."
}

Success Response (200):
{
  "success": true,
  "data": {
    "_id": "507f1f77bcf86cd799439011",
    // ... updated client data
    "updatedAt": "2024-04-01T00:00:00.000Z"
  }
}

JavaScript Example:
const updates = {
  status: 'active',
  contactPerson: {
    name: 'Updated Contact Name',
    title: 'New Title',
    email: 'updated@company.com'
  }
};

const response = await fetch(`/api/v1/clients/${clientId}`, {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${accessToken}`
  },
  body: JSON.stringify(updates)
});

üóëÔ∏è 5. Delete Client

Endpoint: DELETE /api/v1/clients/:id
Headers: Authorization: Bearer {access_token}

Success Response (200):
{
  "success": true,
  "data": {},
  "message": "Client removed successfully"
}

JavaScript Example:
if (confirm('Are you sure you want to delete this client?')) {
  const response = await fetch(`/api/v1/clients/${clientId}`, {
    method: 'DELETE',
    headers: {
      'Authorization': `Bearer ${accessToken}`
    }
  });

  if (response.ok) {
    // Remove from UI
    document.getElementById(`client-${clientId}`).remove();
    showNotification('Client deleted successfully', 'success');
  }
}

üñºÔ∏è 6. Client Logo Upload

Upload client logos using the upload endpoint:

Endpoint: POST /api/v1/upload/single
Headers: Authorization: Bearer {access_token}
Body: FormData

FormData fields:
- file: (logo image file)
- entityType: "client"
- entityId: "507f1f77bcf86cd799439011"
- imageType: "logo" (automatically sets logo field)

JavaScript Example:
const formData = new FormData();
formData.append('file', logoFile);
formData.append('entityType', 'client');
formData.append('entityId', clientId);
formData.append('imageType', 'logo');

const response = await fetch('/api/v1/upload/single', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${accessToken}`
  },
  body: formData
});

// Logo will be automatically updated in client record
const result = await response.json();
if (result.success) {
  // Update UI with new logo
  document.getElementById('client-logo').src = result.data.url;
}

üîß Field Reference

Required Fields:
- name: string (unique company/client name)

Optional Fields:
- logo: string (URL to client logo)
- industry: string (client's business industry)
- website: string (valid URL)
- description: string (max 1000 characters)
- contactPerson: object {
    name: string,
    title: string,
    email: string (valid email),
    phone: string (valid phone number)
  }
- address: object {
    street: string,
    city: string,
    state: string,
    zipCode: string,
    country: string
  }
- status: enum ['prospect', 'active', 'inactive', 'completed'] (default: 'prospect')
- isActive: boolean (default: true)
- projectHistory: array of ObjectIds (references to Project)

Status Definitions:
- prospect: Potential client, not yet engaged
- active: Currently working with this client
- inactive: Temporarily not working together
- completed: All projects finished, relationship ended

Common Use Cases:
1. Client portfolio: Display past and current clients
2. CRM system: Manage client relationships and contacts
3. Project assignment: Link projects to specific clients
4. Business development: Track prospects and active clients
5. Testimonials: Feature satisfied clients
6. Contact management: Store and access client contact information
