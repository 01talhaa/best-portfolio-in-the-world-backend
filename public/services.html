How to use these Service endpoints in your app/website:

🛠️ 1. Get All Services

Endpoint: GET /api/v1/services
Headers: Authorization: Bearer {access_token}

Success Response (200):

      "name": "Web Development",

      "icon": "https://res.cloudinary.com/example/image/upload/v1234567890/web-dev-icon.svg",
      "featured": true,
      "category": "Web Development",
      "tags": ["React", "Node.js", "MongoDB", "JavaScript"],
      "images": [
        "https://res.cloudinary.com/example/image/upload/v1234567890/web-dev-1.jpg",
        "https://res.cloudinary.com/example/image/upload/v1234567890/web-dev-2.jpg"
      ],
      "videos": ["https://youtube.com/watch?v=demo"],
      "pricing": {
        "type": "project",
        "basePrice": 5000,
        "currency": "USD",
        "description": "Starting from $5,000 per project"
      },
      "duration": "2-8 weeks",
      "deliverables": ["Responsive Website", "Admin Panel", "Documentation", "Training"],

      "updatedAt": "2024-03-01T00:00:00.000Z"
    }
  ],

JavaScript Example:
const accessToken = localStorage.getItem('accessToken');
const response = await fetch('/api/v1/services', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
});
const data = await response.json();

// Display services in grid layout
data.data.forEach(service => {
  const serviceCard = document.createElement('div');
  serviceCard.className = 'service-card';
  serviceCard.innerHTML = `
    
    
${service.name}

    
${service.shortDescription}

    

      ${service.tags.map(tag => `${tag}`).join('')}
    

    
${service.pricing.description}

  `;
  document.getElementById('services-grid').appendChild(serviceCard);
});

➕ 2. Create New Service

Endpoint: POST /api/v1/services
Headers: 
  Content-Type: application/json
  Authorization: Bearer {access_token}

Request Body:
{
  "name": "Mobile App Development",
  "description": "Native and cross-platform mobile application development for iOS and Android...",
  "shortDescription": "iOS & Android apps with React Native and Flutter", // optional
  "icon": "https://example.com/mobile-icon.svg", // optional
  "featured": false, // optional, default: false
  "category": "Mobile Development",
  "tags": ["React Native", "Flutter", "iOS", "Android"], // optional
  "images": ["https://example.com/mobile-dev.jpg"], // optional
  "videos": ["https://youtube.com/watch?v=mobile-demo"], // optional
  "pricing": { // optional
    "type": "project",
    "basePrice": 8000,
    "currency": "USD",
    "description": "Starting from $8,000 per app"
  },
  "duration": "3-6 months", // optional
  "deliverables": ["iOS App", "Android App", "Backend API", "Documentation"], // optional
  "prerequisites": ["App requirements", "Design mockups", "API specifications"] // optional
}

Success Response (201):
{
  "success": true,
  "data": {
    "_id": "507f1f77bcf86cd799439012",
    "name": "Mobile App Development",
    // ... all service fields
    "createdAt": "2024-04-01T00:00:00.000Z",
    "updatedAt": "2024-04-01T00:00:00.000Z"
  }
}

JavaScript Example:
const serviceData = {
  name: 'Mobile App Development',
  description: 'Custom mobile applications for iOS and Android...',
  category: 'Mobile Development',
  tags: ['React Native', 'Flutter'],
  pricing: {
    type: 'project',
    basePrice: 8000,
    currency: 'USD'
  }
};

const response = await fetch('/api/v1/services', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${accessToken}`
  },
  body: JSON.stringify(serviceData)
});

🔍 3. Get Service by ID

Endpoint: GET /api/v1/services/:id
Headers: Authorization: Bearer {access_token}

Success Response (200):
{
  "success": true,
  "data": {
    "_id": "507f1f77bcf86cd799439011",
    "name": "Web Development",
    // ... all service fields
  }
}

const serviceId = '507f1f77bcf86cd799439011';

  headers: {
    'Authorization': `Bearer ${accessToken}`

// Create detailed service page

${service.data.category}

  

  

    
${service.data.description}

    
${service.data.pricing?.description || 'Contact for pricing'}

        ${service.data.deliverables?.map(item => `
  • ${item}
    • `).join('') || ''}
      

    

  

`;

📂 4. Get Services by Category

Endpoint: GET /api/v1/services/category/:category
Headers: Authorization: Bearer {access_token}

Available Categories:
- Web Development
- Mobile Development  
- UI/UX Design
- Property Management
- Real Estate
- Software Development
- Consulting
- Other

Success Response (200):
{
  "success": true,
  "data": [
    // Array of services in the specified category
  ],
  "count": 3,
  "category": "Web Development"
}

JavaScript Example:
const category = 'Web Development';
const response = await fetch(`/api/v1/services/category/${encodeURIComponent(category)}`, {
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
});
const data = await response.json();

// Create category section
document.getElementById('category-services').innerHTML = `
  
${data.category} Services (${data.count})

  

    ${data.data.map(service => `
      

        
${service.name}

        
${service.shortDescription}

      

    `).join('')}
  

`;

✏️ 6. Update Service

Endpoint: PUT /api/v1/services/:id
Headers: 
  Content-Type: application/json
  Authorization: Bearer {access_token}

Request Body: (include only fields you want to update)
{
  "featured": true,
  "pricing": {
    "type": "hourly",
    "basePrice": 150,
    "currency": "USD",
    "description": "$150/hour for consultation"
  },
  "tags": ["React", "Node.js", "TypeScript", "PostgreSQL"]
}

Success Response (200):
{
  "success": true,
  "data": {
    "_id": "507f1f77bcf86cd799439011",
    // ... updated service data
    "updatedAt": "2024-04-01T00:00:00.000Z"
  }
}

JavaScript Example:
const updates = {
  featured: true,
  pricing: {
    type: 'hourly',
    basePrice: 150,
    currency: 'USD'
  }
};

const response = await fetch(`/api/v1/services/${serviceId}`, {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${accessToken}`
  },
  body: JSON.stringify(updates)
});

🗑️ 7. Delete Service

Endpoint: DELETE /api/v1/services/:id
Headers: Authorization: Bearer {access_token}

Success Response (200):
{
  "success": true,
  "data": {},
  "message": "Service removed successfully"
}

JavaScript Example:
const response = await fetch(`/api/v1/services/${serviceId}`, {
  method: 'DELETE',
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
});

if (response.ok) {
  // Remove from UI
  document.getElementById(`service-${serviceId}`).remove();
}

📸 8. Service Images Upload

Upload service images using the upload endpoint:

Single Image (Icon or Gallery):
POST /api/v1/upload/single

FormData fields:
- file: (image file)
- entityType: "service"
- entityId: "507f1f77bcf86cd799439011"
- imageType: "icon" or "gallery" (default: "gallery")

Multiple Images (Gallery):
POST /api/v1/upload/multiple

FormData fields:
- files: (multiple image files)
- entityType: "service"
- entityId: "507f1f77bcf86cd799439011"

JavaScript Example:
// Upload service icon
const iconFormData = new FormData();
iconFormData.append('file', iconFile);
iconFormData.append('entityType', 'service');
iconFormData.append('entityId', serviceId);
iconFormData.append('imageType', 'icon');

await fetch('/api/v1/upload/single', {
  method: 'POST',
  headers: { 'Authorization': `Bearer ${accessToken}` },
  body: iconFormData
});

// Upload gallery images
const galleryFormData = new FormData();
galleryFiles.forEach(file => galleryFormData.append('files', file));
galleryFormData.append('entityType', 'service');
galleryFormData.append('entityId', serviceId);

await fetch('/api/v1/upload/multiple', {
  method: 'POST',
  headers: { 'Authorization': `Bearer ${accessToken}` },
  body: galleryFormData
});

🔧 Field Reference

Required Fields:
- name: string (unique)
- description: string
- category: enum ['Web Development', 'Mobile Development', 'UI/UX Design', 'Property Management', 'Real Estate', 'Software Development', 'Consulting', 'Other']

Optional Fields:
- shortDescription: string (max 200 chars)
- icon: string (URL to service icon/image)
- featured: boolean (default: false)
- tags: array of strings
- images: array of strings (URLs to service gallery images)
- videos: array of strings (URLs to service demo videos)
- pricing: object { type, basePrice, currency, description }
- duration: string (estimated completion time)
- deliverables: array of strings (what client receives)
- prerequisites: array of strings (what client needs to provide)

Pricing Types:
- project: Fixed price per project
- hourly: Hourly rate
- monthly: Monthly retainer
- custom: Custom pricing (contact for quote)

Common Use Cases:
1. Services page: Display all available services
2. Category filtering: Show services by type
3. Featured showcase: Highlight main services
4. Service details: Individual service pages
5. Pricing calculator: Dynamic pricing based on requirements
6. Contact forms: Service-specific inquiry forms
